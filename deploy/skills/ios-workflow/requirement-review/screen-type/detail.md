# 상세 화면

단일 리소스를 표시하는 화면(상품 상세, 게시글 등)에 추가 적용한다. 공통 iOS 엣지케이스(다크모드·Dynamic Type·SafeArea·빈상태·딥링크·백그라운드복귀·회전)는 step-1 본체에서 함께 점검한다.

## 존재하지 않는 / 권한 없는 리소스

- 삭제되었거나 존재하지 않는 리소스에 접근 시 처리가 정의되어 있는가? (에러 화면·뒤로가기·안내)
- 딥링크/푸시로 전달된 ID가 유효하지 않을 때 처리가 정의되어 있는가?
- 권한이 없는 리소스(비공개·차단)에 접근 시 처리가 정의되어 있는가?

## 진입 경로

- 이 화면에 진입하는 경로가 여럿인가? (목록 탭 / 딥링크 / 푸시 / 검색) 각 경로에서 필요한 선행 데이터가 다른가?
- 딥링크로 상세에 바로 진입했을 때, 뒤로가기 스택이 자연스러운가? (상위 목록을 합성해 넣을지)
- 백그라운드 복귀 시 상세 데이터를 갱신하는가, 캐시를 유지하는가?

## 콘텐츠 가변성

- 타이틀·설명이 매우 길 때(Dynamic Type 최대 포함) 말줄임/스크롤/줄바꿈 처리가 정의되어 있는가?
- 이미지·미디어가 없을 때의 플레이스홀더가 있는가?
- 공유(`ShareLink`)·외부 링크가 필요한 화면인가?

## 액션

- 좋아요·북마크 등 즉시 반영(옵티미스틱 업데이트)이 필요한가? 실패 시 롤백 처리는?
- 파괴적 액션(삭제)에 확인 다이얼로그가 필요한가?
