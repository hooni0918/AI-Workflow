# 폼 화면

사용자 입력이 주요 인터랙션인 화면에 추가 적용한다. 공통 iOS 엣지케이스(다크모드·Dynamic Type·SafeArea·빈상태·딥링크·백그라운드복귀·회전)는 step-1 본체에서 함께 점검한다.

## 포커스 / 키보드

- 화면 첫 접근 시 메인 입력 필드에 포커스(`@FocusState`·`focused`)가 필요한가?
- 키보드가 올라올 때 입력 필드가 가려지지 않는가? (스크롤·`safeAreaInset`·키보드 회피)
- 키보드 타입(이메일·숫자·전화)·return 키·자동완성·secure 입력이 필드별로 정의되어 있는가?
- 키보드 dismiss 동작(스크롤·바깥 탭·Done 버튼)이 정의되어 있는가?

## 입력 유효성 검증

- 필수/선택 필드 구분이 되어 있는가?
- 각 필드의 타입과 포맷이 정의되어 있는가? (문자열 길이, 숫자 범위, 날짜 형식 등)
- 클라이언트 검증과 서버 검증의 범위가 구분되어 있는가?
- 검증 실패 시 에러 메시지·인라인 표시 위치가 정의되어 있는가?

## 이탈 보호 / 상태 보존

- 뒤로가기·스와이프 dismiss 시 입력값이 유지되는가, 초기화되는가? 확인 다이얼로그가 필요한가?
- 앱이 백그라운드로 갔다가 복귀하면 입력 중이던 값이 유지되는가?
- 입력 도중 앱이 종료(메모리 회수)되면 복구가 필요한가?

## 제출

- 제출 실패 시 입력값이 유지되는가?
- 제출 버튼 더블 탭·중복 제출 방지가 필요한가? (제출 중 비활성화·로딩)
- 제출 성공 후 화면 전환·토스트·dismiss 동작이 정의되어 있는가?
