# 순차 플로우

여러 화면을 순서대로 거쳐야 하는 플로우(온보딩, 회원가입 단계, 결제 등)에 추가 적용한다. 공통 iOS 엣지케이스(다크모드·Dynamic Type·SafeArea·빈상태·딥링크·백그라운드복귀·회전)는 step-1 본체에서 함께 점검한다.

## 네비게이션

- A에서 B로 보낼 때 push vs sheet vs fullScreenCover 중 무엇이 맞는가? 뒤로가기로 이전 단계에 돌아가야 하면 push, 돌아가면 안 되면 경로 교체(`NavigationPath` 갱신)
- `NavigationStack`/`NavigationPath`로 단계 상태를 한곳에서 관리하는가, 화면마다 분산하는가?
- 중간 단계에 딥링크로 직접 진입할 때 이전 데이터 없음 → 시작 단계로 리다이렉트하는가?
- 스와이프 뒤로가기(interactive pop)를 단계 중간에 허용할지 정의되어 있는가?

## 이탈 / 복귀

- 중간에 이탈했다가 다시 돌아왔을 때 이어서 진행 가능한가? (진행 상태 저장)
- 뒤로가기 시 이전 단계 입력값이 유지되는가?
- 백그라운드 복귀·앱 재실행 시 현재 단계 데이터가 유실되는가, 복구되는가?
- 인증·세션이 플로우 도중 만료되면 어느 단계로 보내는가?

## 진행 상태

- 전체 단계 수·현재 단계를 사용자에게 표시하는가? (프로그레스)
- 마지막 단계 완료 후 플로우 전체를 스택에서 제거(시작점 복귀)하는가?
- 도중 취소 시 확인 다이얼로그·중간 데이터 폐기 처리가 정의되어 있는가?
