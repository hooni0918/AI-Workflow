# Code Review

코드 변경을 살펴보고 고칠 거리를 목록으로 뽑아주는 스킬입니다. 고치지는 않고, 무엇이 문제인지만 짚어 줍니다. 수정할지 말지는 사람이 정합니다.

## 동기

코드 리뷰를 하다 보면 같은 지적을 매번 반복하게 됩니다. "이 이름은 이렇게 짓기로 했잖아요" 같은 말을 사람마다, PR마다 다시 하게 됩니다.

그래서 자주 나오는 지적은 코딩 규칙 문서로 한 번 정리해 두고, AI가 그 문서를 직접 읽어 가며 리뷰하도록 만들었습니다. 규칙으로 적어 둔 것은 사람이 다시 말하지 않아도 AI가 짚어 주고, 규칙에 없는 문제는 AI가 경험 많은 리뷰어의 시선으로 따로 살핍니다.

## 리뷰 방식

먼저 리뷰할 코드와 그 코드가 무엇을 하려던 것인지(PR 설명, 변경 의도)를 읽습니다. 그리고 그 코드에 해당하는 코딩 규칙 문서를 골라 와 대조합니다.

구현 파일을 리뷰할 때는 그에 딸린 테스트도 함께 가져와 살핍니다. 코드만 보고 테스트를 안 보면 둘 사이의 문제를 놓치기 때문입니다.

이미 누군가(사람이든 봇이든) 단 지적은 다시 올리지 않고, 새로 발견한 것만 추려 냅니다. 그 작성자가 실제로 손댄 코드만 보고, 원래 있던 코드의 문제는 건드리지 않습니다.

찾아낸 문제는 심각도(꼭 고쳐야 하는 것, 사소한 것, 제안)로 나눠 정리합니다. 규칙에 근거한 지적이면 어느 규칙 문서를 어겼는지까지 함께 적습니다.

## 쓰임새

같은 리뷰라도 쓰임새가 셋입니다.

### 내 코드 점검

작업 워크플로우 마지막의 품질 점검 단계에서 부릅니다. 코딩 규칙을 영역별로 나눠 여러 검사자가 동시에 보고, 규칙 바깥의 깊은 문제는 더 무거운 검사자가 따로 봅니다. 코드를 만든 사람이 자기 코드를 직접 검사하지 않도록, 만드는 역할과 검사하는 역할을 나눕니다.

### 남의 PR 리뷰

한 번 리뷰한 뒤, 그 결과로 코멘트를 달지 말지는 사용자가 정합니다.

### 새 규칙에 맞춰 정렬

규칙을 새로 바꿨을 때, 기존 코드가 그 규칙에 맞는지만 따로 확인합니다. 이때는 자유로운 리뷰는 빼고 규칙 위반만 모아 줍니다.
