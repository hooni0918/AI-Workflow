# examples (예시 모음)

여기 있는 파일들은 **예시**다. 특정 아키텍처(예: Clean Architecture/DDD)·UI 프레임워크(예: UIKit/SwiftUI)·네이밍을 쓰는 프로젝트가 *어떻게 자기 표준을 정의하는지* 보여줄 뿐, 마스터 표준이 아니다.

마스터(이 디렉토리의 상위 `rules/`·`principles/`)는 iOS 표준을 **정의·강제하는 방법**과 범용 규칙·원칙만 담고, 어떤 아키텍처·UI 프레임워크·레이어·네이밍을 쓸지는 고르지 않는다. 그 구체 사실은 **사용 레포가 공급**한다 — 네 프로젝트의 `.claude/docs/architecture.md`, `layer-rules.json`, `.claude/docs/project-profile.md`.

## 이 폴더를 쓰는 법

- 네 프로젝트의 아키텍처·UI 프레임워크와 가까운 예시를 골라 **참고**한다.
- 예시를 그대로 가져다 쓰지 말고, 네 프로젝트의 `.claude/docs/architecture.md`·`layer-rules.json`·`project-profile.md`에 **네 것으로 정의**해 대체한다.
- 각 파일 맨 위에는 그 파일이 예시임을 알리는 배너가 있다. 배너가 가리키는 아키텍처/프레임워크가 네 것과 다르면, 세부 규칙(레이어 이름·네이밍 접미사·의존 방향)은 네 `.claude/docs`를 1차 출처로 따른다.

## 파일 목록

이 폴더의 각 파일은 한 가지 결정(아키텍처 경계, 모듈 구성, 네이밍, UI 작성 패턴, 화면 전환 등)을 특정 스택에서 어떻게 적은 예시다. 파일명을 보고 네 작업과 가까운 것을 골라 Read한다.

- 어떤 파일도 마스터 규칙으로 인용하지 않는다.
- 예시에 나온 구체 사실(레이어 개수·네이밍 접미사·채택 프레임워크)은 네 프로젝트 사실이 아니다. 네 `.claude/docs`에 정의가 있으면 그것을 따르고, 없으면 사용자에게 묻거나 안전 가정(가정 라벨)으로 진행한다.
