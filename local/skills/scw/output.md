# 감사 결과 형식

검증 결과를 AUDIT_RESULT.md 파일로 작성한다.

## 파일 위치

검증 대상과 같은 디렉토리에 저장. 이미 존재하면 기존 내용에 추가.

## 항목 분류

- **지적** — 현재 문서에 존재하는 문제점. 필수 필드: 위치, 문제, 권장
- **제안** — 없어도 되지만 추가하면 좋겠는 내용. 필수 필드: 내용, 근거

## 판정

각 항목에 판정 필드를 placeholder로 남긴다. 사용자가 채운다:
- `승인`
- `반려 — 사유`
- `질문 — 내용`

## 규칙

- 항목이 없으면 파일을 생성하지 않는다
