# SCW — Skill Creator & Prompt Auditor

현재 런타임의 skill-creator에 자체 원칙을 더하고, 스킬 개선·프롬프트 감사·개선 루프를 묶은 스킬입니다.

## 동기

skill-creator에 제 나름의 원칙을 얹기 위해 감쌌습니다.

- **할 일만 적고 방법은 줄이기**: 스킬에는 "무엇을 하라"만 남기고 "어떻게 하라"는 최소화합니다. 방법을 뺐을 때 AI가 정말 못 하는지를 비교 테스트로 확인한 뒤에만 뺍니다.
- **산출물 양식은 자유롭게**: 예시로 양식을 고정하지 않습니다. 꼭 필요한 항목만 정하고 구체적인 형태는 AI에게 맡깁니다. 양식을 못박으면 더 나은 구조를 만들 기회를 막기 때문입니다.
- **실패 사례는 나중에 모으기**: 스킬을 처음 만들 때는 주의사항을 비워 두고, 실제로 쓰다가 AI가 반복해서 실패하는 패턴이 보이면 그때 더합니다.
- **프롬프트 점검**: 완성된 프롬프트 문서를 명확성, 정합성, 구조, 간결성 관점에서 검증합니다.
- **규칙은 한곳에만**: 규칙을 더할 때, 그게 이 스킬이 할 일인지 다른 곳이 할 일인지 따집니다. 다른 곳 일이면 그곳에 두고 여기에는 한 줄 안내만 남깁니다. 같은 규칙이 두 곳에 적혀 양쪽을 같이 고쳐야 하는 부담을 막기 위해서입니다.
- **문제가 0건이 될 때까지 다시 검증**: 검증에서 문제가 하나라도 남으면 스스로 끝내지 않고 다시 한 바퀴 돕니다. "이 정도면 충분하다"가 다음에 같은 문제로 되돌아와 가이드 신뢰를 무너뜨리기 때문입니다.
